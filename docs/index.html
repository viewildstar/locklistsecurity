SOC2 Control Coverage

Our scan evaluates Microsoft 365 tenant configurations against SOC2 Trust Service Criteria. Each check maps to identity, access, and monitoring requirements.

CC6.1 — Logical Access

MFA for Admin Roles

Conditional Access policies targeting privileged roles must require MFA. We verify policies exist, are enabled, and include MFA in grant controls.

CC6.1 — Logical Access

MFA for All Users

Organization-wide MFA enforcement via Conditional Access. We check for policies that include all users and require MFA before granting access.

CC6.1 — Logical Access

Block Legacy Authentication

Legacy protocols (basic auth, IMAP, POP) bypass MFA. We detect policies that block "Other clients" and legacy client types.

CC6.1 — Logical Access

Privileged Role Assignments

Inventory of directory role assignments. Excessive assignments (>10) flag potential over-provisioning. PIM and least privilege recommended.

CC6.1 — Logical Access

Directory Roles & Membership

Audit of Entra ID directory roles and member counts. Ensures visibility into who holds Global Admin, Privileged Role Admin, and other sensitive roles.

CC6.1 — Logical Access

MFA Registration Coverage

Percentage of users with MFA methods registered. Target ≥95%. We pull from the authentication methods user registration report.

CC6.1 — Logical Access

Per-User Auth Methods

Sample of authentication methods per user. Validates strong methods (Authenticator, FIDO2) for admins and high-risk accounts.

CC7.4 — Monitoring

Sign-In Logs

Access to sign-in logs via Microsoft Graph. Required for detecting failed logins, legacy client usage, and anomalous access patterns.

CC7.4 — Monitoring

Directory Audit Logs

Directory audit logs capture admin actions: role changes, app consent, user modifications. Essential for change management evidence.

CC7.4 — Monitoring

Applied CA Policies in Logs

Sign-in logs include appliedConditionalAccessPolicies. Confirms policy evaluation visibility for audit trail completeness.

Token Flow

# User signs in with Microsoft 365 (Entra ID) # Browser obtains access token via MSAL.js — token never leaves client # User clicks "Run Scan" → token sent to backend for single request # Backend calls Microsoft Graph with delegated token
User → MSAL.js → Access Token → Backend → Microsoft Graph
# Graph endpoints called: /identity/conditionalAccess/policies, /auditLogs/signIns, # /roleManagement/directory/roleAssignments, /reports/authenticationMethods/..., etc.

Microsoft Graph Permissions (Delegated)

Admin consent required. Scoped to read-only operations.

  • AuditLog.Read.All — Sign-in and directory audit logs
  • Directory.Read.All — Users, roles, memberships
  • Policy.Read.All — Conditional Access policies
  • RoleManagement.Read.Directory — Role assignments
  • UserAuthenticationMethod.Read.All — MFA registration, auth methods
  • Organization.Read.All — Tenant info, subscribed SKUs
